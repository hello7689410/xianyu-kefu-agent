# 闲鱼人工客服机器人

## 1.大致介绍

项目使用API：get_accessToken(获取accessToen)，get_item_detial(获取商品信息)，然后建立WebSocket协议与闲鱼建立实时通信连接，将每一次顾客发来的信息和商品信息发给大模型，大模型通过这些信息进行决策生成回复。

**特别说明：本篇文章在后面详细展示了项目的开发过程、思路和抓包过程，非常适合新手上手学习，通过开发过程说明和github上的代码，一步一步构造属于自己的如果客服Agent**

 进行分析，里面的sign生成机制强调说明一下

```python
import hashlib


def generate_sign(t: str, token: str, data: str) -> str:
    """生成 sign（MD5）用于 mtop 登录 token 接口。

    说明：消息体拼接格式为：`{token}&{t}&{app_key}&{data}`。
    """
    app_key = "34839810"
    msg = f"{token}&{t}&{app_key}&{data}"
    md5_hash = hashlib.md5()
    md5_hash.update(msg.encode("utf-8"))
    return md5_hash.hexdigest()
```

这里的token是Cookie里面的_m5_h5_tk,直接复制下来即可。

### 3.WebSocket通信机制

#### 建立连接

客户端---ackDiff(同步请求)--->服务器

#### 处理收到的消息

收到的消息是json形式，我们要反序列化提取出真正的聊天信息

利用下面这个

```python
sync_items = parsed.get("body", {}).get("syncPushPackage", {}).get("data", [])
```

得到二进制数据，需要解码

```
eyJjaGF0VHlw··········nNlc3Npb25JZCI6IjU5MTUwOTU3OTg3In0=  #太长我没有全部列出来
```

里面包含base64编码，但又包含二进制编码，如果直接使用base64->utf-8编码，结果中会有乱码

```
2218894644927@goofish59150957987@goofish4025382918959.PNM   erhuihiwehie /{"contentType":1,"text":{"text":"rhuihiwehie"}}	 
```

我们使用MessagePack方式解码，自定义的解码方式

MessagePack:先开始读取类型字节，判断数据类型，然后决定按什么长度读取，在递归解析镶嵌结构

递归解析镶嵌：

<img src="https://gitee.com/zlinping/tuku2025/raw/master/image-20260326221920444.png" alt="image-20260326221920444" style="zoom:50%;" />

解码后将真正有用的数据提取出来,会在后面发送信息时会用的到

```python
# 提取出发送方信息
sender_info = decoded_msg.get(1, {})
details = sender_info.get(10, {})
cid = sender_info.get(2)  # 聊天ID
sender_id = details.get("senderUserId")  # 发送者ID
sender_nickname = details.get("reminderTitle")  # 昵称
sender_content = details.get("reminderContent")  # 消息内容
print(
    f"消息发送方ID: {sender_id}, 昵称: {sender_nickname}, 消息内容: {sender_content}, cid: {cid}"
)
```

在项目主目录下创建Dockerfile

<img src="https://gitee.com/zlinping/tuku2025/raw/master/image-20260327202351847.png" alt="image-20260327202351847" style="zoom:67%;" />

```
FROM python:3.12-slim	#使用官方的Python:3.12-slim运行环境

WORKDIR /app		#设置工作目录，将当前目录改为/app

# 减少运行时缓存占用

    PIP_NO_CACHE_DIR=1 \

```
